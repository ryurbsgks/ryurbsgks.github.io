Sequelize

1. ORM(객체 관계 매핑)

ORM(Object-Relational Mapping; 객체 관계 매핑)은 데이터를 객체 간의 관계를 바탕으로 SQL문으로 만들어서 객체와 관계형 데이터베이스의 데이터를 연결해주는 프로그래밍 기법이다.

2. Sequelize

Sequelize는 Node.js 기반의 ORM이다.

Sequelize에서 제공하는 대부분의 메서드는 비동기로 기본적으로 Promise를 제공하며 async와 await도 사용 가능하다.

3. Sequelize 설치

npm install --save sequelize npm i sequelize

4. Sequelize CLI 설치

CLI는 마이그레이션(Migrations) 및 프로젝트 부트스트랩(Project bootstrapping)에 대한 지원을 제공한다.

npm install --save-dev sequelize-cli

Project bootstrapping

프로젝트 초기 단계를 자동으로 설정할 수 있도록 도와주는 폴더 및 파일

npx sequelize-cli init

성공적으로 완료되면 config, models, migrations, seeders 폴더가 생성된다.

config

데이터베이스에 연결하는 방법을 CLI에 알려주는 config.json 파일을 포함하는 폴더

만약 해당 데이터베이스가 없다면 db:create 명령어로 config/config.json 파일의 해당 데이터베이스를 생성한다.

sequelize db:create

models

프로젝트의 모든 모델 파일을 포함하는 폴더

migrations

모든 마이그레이션 파일을 포함하는 폴더

seeders

모든 시드 파일을 포함하는 폴더

5. 모델 생성

npx sequelize-cli model:generate --name <model_name> --attributes <field_name>:<data_types>,<field_name>:<data_types>

모델은 엔티티를 객체로 표현한 형태로 데이터 구조를 기술하고 데이터에 수행할 수 있는 명령의 모음을 의미한다.

id, createdAt, updatedAt 필드는 자동으로 생성되며 생성된 모델은 모델이름의 파일명으로 models폴더에 위치한다.

6. 마이그레이션

IT에서 마이그레이션은 데이터나 소프트웨어를 다른 시스템으로 이동하는 것을 말한다.

Sequelize에서 마이그레이션은 마이그레이션 실행(up) 및 취소(down) 2가지 기능을 가지는 JS파일이다.

npx sequelize-cli db:migrate

실질적으로 데이터베이스에 테이블을 생성하는 과정으로 테이블명의 기본설정은 모델명의 복수형으로 나온다.
만약 테이블명을 변경하고 싶으면 migrations 폴더에 해당 모델명 파일의 up, down의 createTable, dropTable의 테이블명을 변경한 후 마이그레이션을 진행 하면 된다.

db:migrate 명령어는 SequelizeMeta(현재 데이터베이스에서 실행된 마이그레이션을 기록하는 데 사용되는 테이블)를 확인 후 실행되지 않은 마이그레이션 파일이 있으면 마이그레이션을 진행한다.
따라서 마이그레이션 변동이 있으면 db:migrate을 진행해야 한다.

7. 마이그레이션 취소

db:migrate:undo 명령어는 가장 최근 마이그레이션을 되돌린다.

npx sequelize-cli db:migrate:undo

db:migrate:undo:all 명령어는 모든 마이그레이션을 되돌려 초기 상태로 되돌린다.

npx sequelize-cli db:migrate:undo:all

--to 옵션을 사용하면 특정 마이그레이션을 되돌린다.

npx sequelize-cli db:migrate:undo:all --to XXXXXXXXXXXXXX-create-<model_name>.js
