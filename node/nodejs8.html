Express

1. Express

Express는 Node.js 환경에서 웹 서버 또는 API 서버를 제작하기 위해 사용되는 인기 있는 프레임워크이다.

2. Express 설치

Express는 해당 디렉토리의 package.json의 dependency 항목에 저장하여 설치를 하기 위해 --save 옵션을 붙여 설치한다.

npm install --save express

만약 Express를 임시로 설치하고 dependency 항목에 추가하지 않으려면 --save 옵션을 생략한다.

npm install express

Express 불러오기

Express 설치가 완료되면 아래와 같이 require() 함수를 사용하여 Express를 불러올 수 있다.

const express = require('express')

웹 서버 만들기

Express를 사용하여 간단한 웹 서버를 만드는 방법으로 기본 틀은 아래와 같다.

const express = require('express')
const app = express()
const port = <portNumber>

app.listen(port, () => {
  console.log("Server Running")
})

3. Express 메서드

Express에는 많은 메서드가 내장되어 있지만 여기에서는 app 객체, req 객체, res 객체의 일부 메서드들만 다룬다.
더 많은 Express 메서드에 대해서 알고 싶다면 Express 공식 API 문서를 참고하자

app 객체

app 객체는 Express에서 내보낸 최상위 express() 함수를 호출하여 생성된 객체로 일반적으로 Express 응용 프로그램을 나타낸다.

app.use()

지정된 경로에 지정된 미들웨어 함수를 실행하는 메서드로 미들웨어 함수는 요청된 경로의 기준이 매개변수 <path>와 일치할 때 실행된다.

app.use(<path>, <callback>)

<path>는 미들웨어 기능이 호출되는 경로로 생략 가능하며 생략 시 default 값은 "/" 이다.

<callback>은 미들웨어 기능이 있는 함수가 callback 함수로 올 수 있으며 쉼표로 구분하여 여러개의 미들웨어 함수가 올 수도 있다.

app.METHOD()

지정된 콜백함수를 사용하여 주어진 HTTP 요청을 지정된 경로로 라우팅하는 메서드이다.

app.<METHOD>(<path>, <callback>)

<METHOD>는 소문자로 된 HTTP 요청 메서드가 올 수 있다.

<path>는 미들웨어 기능이 호출되는 경로이다.

<callback>은 미들웨어 기능이 있는 함수가 callback 함수로 올 수 있으며 쉼표로 구분하여 여러개의 미들웨어 함수가 올 수도 있다.

app.all()

app.all() 메서드는 모든 HTTP 요청 메서드와 일치한다는 점을 제외하고는 app.METHOD() 메서드와 비슷하다.

app.all(<path>, <callback>)

<path>와 <callback>은 app.METHOD()의 매개변수와 같다.

app.listen()

지정된 port에 연결을 수신하는 서버를 시작하는 메서드이다.

app.listen(<port>, <callback>)

<port>는 시작하는 서버의 port 번호로 생략되거나 0이면 사용하지 않는 임의의 port를 할당한다.

<callback>은 listen() 메서드에서 사용할 callback 함수로 생략 가능하다.

req 객체

req 객체는 HTTP 요청을 나타내며 요청의 query string, parameters, body, HTTP headers 등에 대한 속성을 가지고 있다.

req.body

요청 body에 제출된 데이터의 key-value 쌍을 포함한다.

기본적으로 undefined이지만 express.json() 또는 express.urlencoded()와 같은 바디 파싱 미들웨어를 사용하여 요청 body에 제출된 데이터를 확인할 수 있다.

req.body
express.json()

Content-Type이 application/json인 요청 body를 파싱하기 위한 바디 파싱 미들웨어이다.

express.urlencoded()

Content-Type이 application/x-www-form-urlencoded인 요청 body 파싱하기 위한 바디 파싱 미들웨어이다.

res 객체

res 객체는 Express 앱이 HTTP 요청을 받을 때 보내는 HTTP 응답을 나타낸다.

res.status()

응답에 대한 HTTP 상태를 설정하는 메서드이다.

res.status(<status>)

<status>는 HTTP 상태 코드를 나타내는 양의 정수가 올 수 있다.

res.send()

HTTP 응답을 보내는 메서드이다.

res.send(<body>)

<body>는 버퍼 객체, 문자열, 객체, 배열, boolean이 올 수 있다.

res.json()

주어진 매개변수를 JSON.stringify()를 사용하여 JSON 문자열로 변환하여 JSON 응답을 보내는 메서드이다.

res.json(<body>)

<body>는 객체, 배열, 문자열, 숫자, null 등 모든 JSON 타입이 될 수 있다.

res.end()

응답 프로세스를 종료하는 메서드로 주로 데이터 없이 응답을 빠르게 종료하는데 사용한다.
만약 데이터로 응답해야 하는 경우는 보통 res.send(), res.json()과 같은 메서드를 사용한다.

res.end()

res.redirect()

주어진 경로 또는 URL로 redirect하는 메서드이다.

res.redirect(<status>, <path>)

<status>는 HTTP 상태 코드를 나타내는 양의 정수로 생략 가능하며 생략 시 302 Found 로 설정된다.

<path>는 경로 또는 URL이 올 수 있다.

res.cookie()

주어진 매개변수로 HTTP Set-Cookie 헤더를 설정하여 쿠키를 생성하는 메서드이다.

res.cookie(<name>, <value>, <options>)

<name>, <value> 매개변수는 각각 cookie의 key-value에 할당된다.

<value>는 문자열 또는 JSON으로 변환된 객체이다.

<options>는 HTTP Set-Cookie 헤더를 설정할 수 있는 객체로 생략 가능하다.
<options>에 올 수 있는 Property는 공식문서를 참고하자

res.clearCookie()

매개변수로 주어진 쿠키를 지우는 메서드이다.

res.clearCookie(<name>, <options>)

<name>은 지우려고 하는 쿠키의 name을 입력한다.

<options>는 res.cookie() 메서드의 options 매개변수와 같으며 생략 가능하다.
