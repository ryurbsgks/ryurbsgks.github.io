Commit Message Template

1. Commit Message Template

Git을 통해 여러 명이 버전관리를 할 때 Commit Message Template을 사용하여 서로 Commit Message의 규칙을 지키면 가독성도 좋고 프로젝트 수행에서 버전관리를 더 원활하게 할 수 있다.

Git은 commit시 commit tempate 옵션에 설정한 템플릿 파일을 보여주는데 이를 통해 commit message를 작성할 때 일정한 스타일을 유지할 수 있다.

2. Tempate 생성 및 설정

.git이 있는 폴더 위치에 .gitmessage.txt 파일을 생성한다.

생성된 .gitmessage.txt 파일에 원하는 템플릿을 작성한다.
아래 예시와 같은 템플릿을 작성해주면 된다.

################
# 타입 : 제목 의 형식
# ex) Feat: 친구검색 기능 추가


# 제목과 본문의 분리를 위한 공백란

################
# 큰제목은 * / 하위 상세 메세지는 - 작성
# * 큰 제목
# - 하위 구현


################
# Feat: 새로운 기능 추가
# Fix: 버그 수정
# Docs: 문서 수정
# Update: 기능 수정 및 추가
# Style: 코드 스타일 변경 (코드 포매팅, 세미콜론 누락 등)
# Design: 사용자 UI 디자인 변경 (CSS 등)
# Refactor: 리팩토링 (Production Code)
# Build: 빌드 파일 수정
# Perf: 성능 개선
# Core: 자잘한 수정이나 빌드 업데이트
# Rename: 파일 혹은 폴더명을 수정만 한 경우
# Remove: 파일을 삭제만 한 경우
################

템플릿 작성이 완료되면 아래 명령어를 통해 commit시 템플릿을 불러오는 설정을 한다.

git config commit.template .gitmessage.txt

만약 모든 프로젝트에서 같은 템플릿을 불러오려면 --global 옵션을 추가한다.

git config --global commit.template .gitmessage.txt
