데이터베이스 정규화

1. 데이터베이스 정규화

데이터베이스 정규화를 통해 불필요한 데이터의 중복을 방지하고 데이터의 삽입, 갱신, 삭제시 발생할 수 있는 각종 이상현상들을 방지할 수 있다.

2. 데이터 중복(Data Redundancy)

데이터베이스 정규화는 데이터 중복을 방지하는 목적을 지닌다.

데이터 중복은 실제 데이터의 동일한 복사본이나 부분적인 복사본을 뜻한다.
이러한 데이터 중복으로 데이터를 복구할 때에 더 수월할 수도 있지만 일관된 자료 처리의 어려움, 저장 공간 낭비, 데이터 효율성 감소등과 같은 문제점을 지닌다.

3. 데이터 무결성(Data Integrity)

데이터베이스 정규화는 데이터 무결성을 강화하는 목적을 지닌다.

데이터 무결성은 데이터의 수명 주기 동안 정확성과 일관성을 유지하는 것을 뜻한다.

정확성

입력된 데이터가 오염되지 않는 것을 말한다.

일관성

입력된 그대로 데이터를 사용할 수 있는 것을 말한다.

4. 이상 현상(Anomaly)

데이터베이스 정규화는 이상 현상을 방지하는 목적을 지닌다.

이상 현상은 기대한 데이터와 다른 이상 현상을 말하며 갱신, 삽입, 삭제 이상이 있다.

갱신 이상(Update Anomaly)

갱신 이상은 여러 레코드에 걸쳐 동일한 데이터가 있을 때 어떤 레코드를 갱신해야 하는지 논리적인 일관성이 없는 경우에 발생한다.

삽입 이상(Insertion Anomaly)

삽입 이상은 데이터를 삽입하지 못하는 경우에 발생한다.

삭제 이상(Deletion Anomaly)

삭제 이상은 데이터의 특정 부분을 지울 때 의도치 않게 다른 부분도 함께 지우는 경우에 발생한다.
