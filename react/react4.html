Props

1. Props

props는 부모 컴포넌트로부터 자식 컴포넌트로 전달된 데이터이다.

props는 읽기 전용으로 어떤 타입의 값도 넣어 전달할 수 있도록 객체의 형태를 가진다.
props가 읽기 전용이 아니면 props를 전달받은 컴포넌트 내에서 props를 수정 시 props를 전달한 상위 컴포넌트의 데이터에 영향을 미칠 수 있게 된다.
이는 개발자가 의도하지 않은 side effect가 생기게 되고 React의 단방향, 하향식 데이터 흐름 원칙에 위배된다.

2. Props 사용 방법

props를 사용하기 위해서는 부모 컴포넌트와 자식 컴포넌트가 존재하고 부모 컴포넌트 안에 자식 컴포넌트가 작성 되어야 한다.

부모 컴포넌트 안에 작성된 자식 컴포넌트에 전달하고자 하는 값(value)과 속성(attribute)을 입력한다.
이때 전달하고자 하는 값은 중괄호({})를 이용하여 감싸주어야 한다.

React 컴포넌트는 JS 함수와 클래스로 props를 함수의 매개변수처럼 전달받는다.
전달받은 props는 객체 형태이고 이 객체의 key-value 쌍은 전달하고자 했던 attribute-value 쌍의 형태를 띠게 된다.
전달받은 props의 value는 Dot notation으로 접근할 수 있다.

function <Parent>() {
  return (
    <ChildComponent <attribute>={<value>} />
  )
}

function <ChildComponent>(props) {
  props.<attribute> {/* expect output : <value> */}
}

전달하고자 하는 값을 매개변수로 전달받을 때 구조 분해 할당으로 받을 수도 있다.

function <ChildComponent>({ <attribute> }) {
  <attribute> {/* expect output : <value> */}
}
