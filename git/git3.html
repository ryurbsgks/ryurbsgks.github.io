Git 설치 및 환경설정 & Github SSH 등록

1. Git 설치

sudo apt install git

설치 확인을 위해서 아래 코드를 입력하여 터미널창에서 git version을 확인한다.

git --version

2. Git 환경설정

Git을 이용할 때 필요한 환경 설정은 사용자 정보와 에디터 설정이다.

사용자 정보 설정

Git을 설치하면 가장 먼저 사용자 이름과 이메일 주소를 설정한다.
설정에 기록된 사용자 이름과 메일 주소를 앞으로 진행할 Git 커밋 내역에 기록한다.

git config --global user.name "<github_user_name>"
git config --global user.email "<github_user_email>"

--global 옵션으로 설정하면 사용자 홈에 저장되므로 git을 설정할 때 처음에 한번만 입력하면 된다.

에디터 설정

Git에서 커밋 메시지를 기록할 때 또는 merge commit 확인 메시지가 나올 때 텍스트 에디터가 열린다.
기본값으로 텍스트 에디터는 vim이 열리는데 vim이 익숙하지 않으면 nano로 변경하여도 좋다.

nano로 변경

git config --global core.editor nano

vim으로 변경

git config --global core.editor vim

3. Github SSH 등록

Github에 SSH를 등록하는 방법과 SSH를 이용해 git clone하는 방법에 대한 설명

SSH 키 생성

아래 명령어를 입력하여 SSH 키 페어를 생성한다.
명령어 입력후 Enter키를 몇 번 입력하면 SSH 키 페어가 생성된다.

ssh-keygen

성공적으로 완료되면 ~/.ssh./ 경로에 id_rsa와 id_rsa.pub이 생성된다.
이 두 파일이 SSH 키 페어이며 id_rsa.pub은 공개키 id_rsa는 개인키이다.

공개키 복사

아래 명렁어를 통해 공개키를 CLI창에 출력한다.

cat ~/.ssh/id_rsa.pub

CLI창에 출력된 공개키를 복사한다.

Github에 공개키 등록

Github에 로그인하여 우측 상단의 프로필 이미지를 클릭하고 Settings탭에 접속한다.

왼쪽의 네비게이션 바에서 SSH and GPG keys탭을 클릭한다.

화면의 우측 상단의 New SSH Key의 초록색 버튼을 클릭한다.

공개키를 구분할 수 있도록 사용자 임의로 Title란에 작성하고 복사한 공개키는 Key란에 입력한다.

Add SSH Key 버튼을 클릭하여 공개키를 등록하고 SSH Key 등록 승인절차가 나오면 비밀번호를 입력 후 SSH Key 등록을 승인해 준다.

이제 원하는 Repository clone하고 싶을 때 Code 버튼에서 HTTPS가 아닌 SSH탭의 주소를 복사해서 clone 해준다.
