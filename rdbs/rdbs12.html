MQL 쿼리 연산자

1. MQL 쿼리 연산자

MQL(MongoDB Query Language) 쿼리 연산자는 데이터베이스 내에서 데이터를 찾는 방법을 제공해준다.

2. 비교 연산자

비교 연산자를 사용하면 특정 범위 내에서 데이터를 찾을 수 있다.

$eq

지정된 값과 같은 값을 찾는다.

{<field>: {$eq: <value>}}

$ne

지정된 값과 같지 않은 값을 찾는다.

{<field>: {$ne: <value>}}

$gt

지정된 값보다 큰 값을 찾는다.

{<field>: {$gt: <value>}}

$lt

지정된 값보다 작은 값을 찾는다.

{<field>: {$lt: <value>}}

$gte

지정된 값보다 크거나 같은 값을 찾는다.

{<field>: {$gte: <value>}}

$lte

지정된 값보다 작거나 같은 값을 찾는다.

{<field>: {$lte: <value>}}

$in

지정된 배열의 값중 하나 이상 포함된 배열이 있는 값을 찾는다.

{<field>: {$in: [<value>, ...]}}

$nin

지정된 배열의 값과 일치하지 않는 배열을 찾는다.
만약 배열의 요소가 여러개일때 요소중 하나라도 포함되어 있으면 $nin 연산자는 적용되지 않는다.

{<field>: {$nin: [<value>, ...]}}

3. 논리 연산자

$and, $or, $nor 연산자는 작동할 절의 배열 앞에 위치하며 $not 연산자는 배열 구문이 필요하지 않다.

$and

지정된 모든 쿼리절을 충족하는 도큐먼트를 반환한다.

{$and: [{<query>}, {<query>}, ...]}

$or

쿼리절 중 하나라도 일치하는 도큐먼트가 있다면 해당 도큐먼트를 반환한다.

{$or: [{<query>}, {<query>}, ...]}

$nor

모든 절과 일치하지 않는 도큐먼트를 반환한다.

{$nor: [{<query>}, {<query>}, ...]}

$not

뒤의 조건을 만족하지 않는 모든 도큐먼트를 반환한다.

{$not: {<query>}}

4. 배열 연산자

배열 연산자를 쓰지 않고 배열 필드를 쿼리하는 경우 배열을 지정한다면 해당 배열과 정확히 일치하는 배열을 가진 도큐먼트를 찾는다.
또한 문자열로 요소를 지정한다면 해당 요소가 배열에 포함된 모든 도큐먼트를 찾는다.
따라서 상황에 맞게 배열 연산자를 사용해야 한다.

$all

배열 요소의 순서와 상관없이 지정된 요소가 모두 포함된 도큐먼트를 반환한다.

{<field>: {$all: [<value>, ...]}}

$size

지정된 배열 필드가 주어진 길이와 정확히 일치하는 모든 도큐먼트 반환한다.

{<field>: {$size: <value>}}
