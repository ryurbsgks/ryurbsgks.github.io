Git 명령어

1. Init

현재 디렉토리를 Git Repository로 변환하여 Git의 관리를 받게하거나 기존 Git Repository를 초기화하는 명령어

git init

2. Clone

코드를 수정하기 위해서 Remote Repository에서 Local Repository로 가져오는 명령어

git clone <Repository 주소>

3. Log

현재까지 Commit 된 로그들을 터미널 창에서 확인하는 명령어로 터미널 창을 종료하는 방법은 "q" 를 입력하면 된다.

git log

4. Status

Local Repository의 Staging area와 Unstaged 상태 그리고 현재 브랜치를 확인할 수 있는 명령어

git status

5. Add

작업에 들어간 파일들을 git의 관리하에 있는 영역(Staging area)으로 올려주는 명령어

git add <file_name>

<file_name>에는 현재 위치에서부터 변경된 파일 경로의 상대경로도 같이 입력해준다.(현재 위치가 같으면 경로는 쓰지 않고 파일명만 써도 된다)

git add .

위 명령어는 Staging area에 Unstaged 상태인 모든 파일을 올려주는 명령어이다.

6. Restore

Commit 되지 않은 Local Repository의 변경 사항을 폐기하는 명령어

git restore <file_name>

<file_name>은 Add와 같다.

7. Commit

Commit을 하기 위해서는 파일들이 Staging area에 있어야 하며 Commit 명령어를 입력하면 설정된 에디터로 Commit 메세지를 작성할 수 있다.

git commit

m 옵션은 vim이나 nano같은 에디터에서 별도의 메세지를 작성할 필요없이 인라인 형식으로 메세지를 작성하게 도와주는 옵션이다.

git commit -m "<message>"

Commit 기본 에디터 변경

Commit 메세지를 작성하는 에디터를 변경하는 방법

vim으로 변경

git config core.editor "vim"

global 옵션은 에디터를 전역으로 설정할 수 있다.

git config --global core.editor "vim"

nano로 변경

git config core.editor "nano"

global 옵션은 에디터를 전역으로 설정할 수 있다.

git config --global core.editor "nano"

8. Reset

Remote Repository에 업로드되지 않고 Local Repository에만 Commit 해 놓은 기록이라면 Reset 명령어를 통해서 Commit 을 취소할 수 있다.

git reset --<option> <commit_id>

<commit_id>는 git log를 통해서 Commit의 id를 직접 입력하거나 HEAD를 사용한다.(Commit의 id는 7자리까지만 입력해도 된다)
HEAD는 연속된 ^의 shortcut으로 HEAD3는 HEAD^^^와 같다.(가장 최근의 Commit을 삭제하려면 HEAD^를 사용한다)

<option>은 선택사항이며 옵션의 종류에는 hard, mixed, soft가 있다.

hard

hard는 돌아가려는 Commit 다음의 모든 내용을 삭제한다.

mixed

mixed는 default값으로 돌아가려는 Commit으로 돌아가고 그 이후의 변경된 내용은 지워지지 않고 해당 내용의 인덱스는 초기화된다.

soft

soft는 돌아가려는 Commit으로 돌아가지만 그 이후의 변경된 내용은 지워지지 않고 해당 내용의 인덱스도 그대로 있다.

9. Push

Commit된 내용을 Remote Repository에 올려주는 명령어

git push <Remote_repository> <branch_name>

<Remote_repository>는 Remote repository의 단축이름이며 <branch_name>는 branch 명이다.

10. Pull

Remote Repository의 내용을 Local Repository로 가져오는 명령어

git pull <Remote_repository> <branch_name>

<Remote_repository>는 Remote repository의 단축이름이며 <branch_name>는 branch 명이다.

11. Remote

등록된 Remote repository의 단축이름을 확인하는 명령어

git remote

v 옵션은 등록된 Remote repository의 단축이름과 URL까지 함께 보여 주는 옵션이다.

git remote -v

12. Remote add

Local Repository를 Remote Repository와 연결하는 명령어

git remote add <Remote_repository> <Repository 주소>

<Remote_repository>는 Remote repository의 단축이름이다.

13. Branch

새로운 브랜치를 생성하거나 브랜치 목록을 확인 또는 삭제하는 명령어

git branch <new_branch_name>

<new_branch_name> 브랜치를 생성한다.

git branch

브랜치 목록을 확인한다.

git branch -v

v 옵션은 브랜치 목록과 각 브랜치의 최근 commit을 확인하는 옵션이다.

git branch -d <branch_name>

d 옵션은 <branch_name> 브랜치를 삭제하는 옵션이다.

git branch -D

D 옵션은 병합하지 않은 브랜치를 강제 삭제하는 옵션이다.

14. Switch

브랜치를 전환하는 명령어

git switch <branch_name>

c 옵션은 새로운 브랜치를 생성 후 해당 브랜치로 전환하는 옵션이다.

git switch -c <branch_name>

15. Checkout

브랜치를 전환하는 명령어

git checkout <branch_name>

b 옵션은 새로운 브랜치를 생성 후 해당 브랜치로 전환하는 옵션이다.

git checkout -b <branch_name>

16. Merge

브랜치를 병합하는 명령어

git merge <branch_name>

현재 위치의 브랜치에 <branch_name> 브랜치를 병합한다.

17. Stash

아직 마무리하지 않은 작업을 스택에 잠시 저장할 수 있도록 하는 명령어

git stash
