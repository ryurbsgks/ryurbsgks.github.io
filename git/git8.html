.gitignore

1. .gitignore

.gitignore 파일이란 Git 버전 관리에서 제외할 목록을 지정하는 파일로 Git으로 프로젝트 버전관리를 할 때 특정 파일을 추적하지 않게 .gitignore 파일에 입력 해주면 된다.

.gitignore 파일은 프로젝트 최상위 디렉토리에 만들어서 버전관리에 제외할 파일을 입력하고 저장하면 된다.

버전관리 중인 파일 제외

.gitignore 파일 작성 전 버전관리에서 제외해야 할 파일이 이미 버전관리 중이라면 아래 명령어를 사용하여 버전관리에서 제외할 수 있다.

git rm --cached <fileName>

--cached 옵션을 사용하면 원격 저장소에 있는 파일만 삭제하며 --cached 옵션을 사용하지 않으면 원격 저장소와 로컬 저장소에 있는 파일을 모두 삭제한다.

만약 모든 파일과 그 하위 파일까지 버전관리에서 제외하려면 아래 명령어를 사용한다.

git rm -r --cached .

.gitignore 목록에 있어도 버전관리 해야 하는 경우

만약 .gitignore 목록에 있어도 버전관리를 해야하는 경우 -f 옵션을 사용하여 버전관리를 하면 된다.

git add -f <fileName>

특정 폴더에 .gitignore 적용

.gitignore에 설정한 목록은 하위 폴더에도 적용되지만 만약 특정 폴더에 다른 .gitignore을 적용해야 하는 경우 해당 폴더에 .gitignore를 만들어 .gitignore을 적용할 수 있다.

2. .gitignore 패턴 형식

.gitignore 파일을 작성하는 패턴 형식은 아래와 같다.

빈 줄은 파일과 일치하지 않으므로 가독성을 위한 구분 기호로 사용할 수 있다.

#으로 시작하는 줄은 주석 역할을 한다.
만약 #으로 시작하는 패턴의 경우 첫 번째 # 앞에 백슬래쉬(\)를 넣는다.

뒤에 오는 공백은 백슬래쉬(\)를 사용하지 않는 한 무시된다.

!는 패턴을 부정하는 의미로 이전 패턴에서 제외된 모든 일치 파일을 다시 포함한다.
만약 해당 파일의 상위 디렉토리가 제외된 경우 파일을 다시 포함할 수는 없다.
만약 !로 시작하는 패턴의 경우 첫 번째 ! 앞에 백슬래쉬(\)를 넣는다.

/는 디렉토리 구분자로 사용되며 패턴의 시작, 중간, 끝에 사용할 수 있다.

패턴 끝에 / 기호가 있으면 패턴은 디렉토리만 일치하고 그렇지 않으면 패턴은 파일과 디렉토리 모두와 일치한다.

*는 /를 제외한 모든 항목과 일치한다.

**는 작성되는 위치에 따라 의미하는 바가 달라진다.
** 다음에 /가 있으면(**/) 모든 디렉토리에서 일치한다.
** 이전에 /가 있으면(/**) 내부의 모든 항목과 일치한다.
** 이전과 다음에 /가 있으면(/**/) 0개 이상의 디렉토리와 일치한다.
예를 들어 a/**/b 는 a/b, a/x/b, a/x/y/b 등과 일치한다.

3. gitignore.io

사용하는 언어나 프레임워크마다 .gitignore에 추가해야할 목록이 있으며 이를 일일이 기억하는 것은 매우 귀찮고 어려운 일이다.
이런 귀찮고 어려운 일을 대신해 주는 gitignore.io라는 사이트가 있으니 활용하는 것도 방법이다.
