Client-Server 아키텍처

1. Client-Server 아키텍처

상품 정보 같은 리소스가 존재하는 곳과 리소스를 사용하는 앱을 분리시킨 것을 Client-Server 아키텍처 또는 2티어 아키텍처라고 부른다.

일반적으로 서버는 리소스를 전달해 주는 역할만 담당하며 리소스를 저장하는 공간을 별도로 마련해 두는데 이 공 간을 데이터베이스라고 부른다.
이처럼 기존 2티어 아키텍처에 데이터베이스가 추가된 형태를 3티어 아키텍처라고 부른다.

2. Client-Server 통신

Client-Server 아키텍처에서는 Client와 Server가 HTTP라는 프로토콜을 이용해서 통신을 하며 주소(URL, URI)를 통해 접근할 수 있다.
HTTP를 이용해 주고받는 메시지는 HTTP 메시지라고 부른다.

프로토콜

프로토콜(Protocol)은 메시지를 주고 받는 양식과 규칙의 체계이다.

3. API

API(Application Programming Interface)는 서버가 클라이언트에게 리소스를 잘 활용할 수 있도록 인터페이스(interface)를 제공해 주는 것을 말한다.
