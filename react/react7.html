useEffect

1. Side Effect

함수 내에서 어떤 구현이 함수 외부에 영향을 끼치는 경우 해당 함수는 Side Effect가 있다고 이야기 한다.

2. Effect Hook

React는 Side Effect를 다루기 위한 Hook인 Effect Hook을 제공한다.

Hook 사용 시 주의할 점

Hook을 사용할 때 다음 두 가지 규칙을 준수해야 한다.

최상위에서만 Hook을 호출한다

항상 React 함수의 최상위(at the top level)에서 Hook을 호출해야 한다.
이 규칙을 따르면 컴포넌트가 렌더링 될 때마다 항상 동일한 순서로 Hook이 호출되는 것이 보장된다.
이러한 점은 React가 useState 와 useEffect 가 여러 번 호출되는 중에도 Hook의 상태를 올바르게 유지할 수 있도록 해준다.

React 함수 내에서 Hook을 호출한다

Hook은 오직 React 함수 내에서 호출해야 한다.
이 규칙을 지키면 컴포넌트의 모든 상태 관련 로직을 소스코드에서 명확하게 보이도록 할 수 있다.

3. useEffect

useEffect는 Effect Hook으로 컴포넌트 내에서 Side effect를 실행할 수 있게 하는 Hook이다.

useEffect 실행 조건

useEffect는 다음과 같이 매번 새롭게 컴포넌트가 렌더링 될 때 실행된다.

컴포넌트 생성 후 처음 화면에 렌더링 될 때

컴포넌트에 새로운 props가 전달되며 렌더링 될 때

컴포넌트에 상태(state)가 바뀌며 렌더링 될 때

useEffect 사용 방법

import { useState } from 'react'

useEffect( () => {
  <동작할 코드>
}, [<value>, <value>, ...])

useEffect의 첫 번째 매개변수는 함수이며 해당 함수 내에서 Side effect를 실행하면 된다.

useEffect의 두 번째 매개변수는 배열로 특별히 종속성 배열이라고 부르며 조건을 담거나 생략할 수 있다.
여기서 조건은 boolean 형태의 표현식이 아닌 어떤 값의 변경이 일어날 때를 의미한다.
따라서 해당 배열엔 어떤 값의 목록이 들어간다.

두 번째 매개변수 배열을 빈 배열로 둘 경우

useEffect( () => {
  <동작할 코드>
}, [])

이때에는 컴포넌트가 처음 생성될 때만 useEffect 함수가 실행된다.

두 번째 매개변수를 생략한 경우

useEffect( () => {
  <동작할 코드>
})

가장 기본 형태의 useEffect로 useEffect 실행 조건일 때 useEffect 함수가 실행된다.
