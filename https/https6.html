Seesion

1. Session

Session이란 웹 사이트의 여러 페이지에 걸쳐 사용되는 사용자 정보를 저장하는 방법을 의미한다.

Session은 서비스가 돌아가는 서버 측에 데이터를 저장하고 세션의 ID를 클라이언트 측에 남겨둔다.
브라우저는 필요할 때마다 이 세션 ID를 이용하여 서버에 저장된 데이터를 사용한다.

Session도 세션의 ID를 클라이언트에 남겨두기 때문에 세션 ID를 Cookie로 전송한다.

2. express-session

express-session은 세션을 대신 관리해 주는 모듈로 Express에서 세션을 다룰 수 있는 공간을 보다 쉽게 만들어준다.

express-session 설치

npm install express-session

session(<options>)

express-session 세션 사용 방법은 session(<options>)을 통해 주어진 옵션으로 세션 미들웨어를 생성한다.
생성된 세션 미들웨어는 응답 객체의 프로퍼티로 Session 이라는 객체를 추가한다.

secret

필수 옵션으로 세션 ID를 쿠키에 sign하는데 사용된다.

값으로는 문자열 또는 배열이 올 수 있으며 배열일 경우 첫 번째 요소만 sign하는데 사용된다.

secret값을 변경하면 기존의 모든 세션이 무효화된다.
만약 세션을 무효화하지 않고 암호를 로테이션하려면 새로운 secret값을 첫 번째 요소로 사용하고 이전 secret값을 이후 요소로 포함하여 배열을 값으로 준다.

resave

세션을 요청하는 동안 세션이 변경되지 않은 경우에도 세션을 세션 저장소에 다시 저장되도록 한다.

기본적으로 true이지만 일반적으로 false를 사용하길 권장한다.

saveUninitialized

세션이 처음 만들어졌을 때 세션을 수정하지 않는다면 세션은 "Uninitialized" 상태이고 이 세션을 세션 저장소에 저장할 것인지 boolean값을 지정한다.

true는 Uninitialized 상태의 세션을 강제로 저장하며 false는 Uninitialized 상태의 세션을 저장하지 않는다.

기본적으로 true이지만 나중에 기본값이 변경되므로 기본값 사용은 더 이상 사용되지 않는다.
false를 선택하면 로그인 세션을 구현하거나, 서버 스토리지 사용을 줄이는 데 유용하다.

store

세션 저장소 인스턴스로 기본적으로 MemoryStore 인스턴스가 사용된다.

cookie

세션 ID에 대한 쿠키 설정 객체로 default는 {path: "/", httpOnly: true, secure: false, maxAge: null} 이다.

cookie.domain

Set-Cookie의 Domain 속성의 값을 지정한다.

기본적으로 도메인이 설정되어 있지 않으며 클라이언트는 쿠키가 현재 도메인에만 적용되는 것으로 간주한다.

cookie.expires & cookie.maxAge

expires는 Set-Cookie의 Expires 속성의 값으로 Date 객체를 지정한다.

기본적으로 expires가 설정되어 있지 않으며 클라이언트는 비영구 쿠키로 간주하고 브라우저가 종료되면 삭제된다.

주의할점으로 expires는 직접 설정하면 안되고 maxAge를 통해서 설정해야 한다.

maxAge는 Set-Cookie의 Expires 속성을 계산할 때 사용할 숫자(milliseconds: 밀리초)를 지정한다.

기본적으로 maxAge는 설정되어 있지 않으며 현재 서버 시간에 maxAge의 밀리초를 추가하여 Expires의 Date 객체로 날짜와 시간을 계산한다.

expires와 maxAge가 모두 설정되어 있으면 cookie 객체의 마지막 항목이 사용된다.

cookie.httpOnly

Set-Cookie의 HttpOnly 속성에 대한 boolean값을 지정한다.

true는 HttpOnly 속성이 설정되고 false는 HttpOnly 속성이 설정되지 않는다.
기본적으로 true이며 HttpOnly 속성이 설정된다.

cookie.path

Set-Cookie의 Path 속성의 값을 지정한다.

기본적으로 도메인의 루트 경로인 "/"로 설정되어 있다.

cookie.sameSite

Set-Cookie의 SameSite 속성의 값이 될 boolean 또는 문자열을 지정한다.

true는 SameSite 속성을 Strict로 설정하고 false는 SameSite 속성을 설정하지 않는다.

"lax"는 SameSite 속성을 Lax로 설정한다.
"none"은 SameSite 속성을 None으로 설정한다.
"strict"는 SameSite 속성을 Strict로 설정한다.

cookie.secure

Set-Cookie의 Secure 속성에 대한 boolean값을 지정한다.

true는 Secure 속성이 설정되고 false는 Secure 속성이 설정되지 않는다.
기본적으로 false이며 Secure 속성이 설정되어 있지 않다.

const express = require("express");
const session = require("express-session")
const app = express();
app.use(session({
  secret: "<secret_key>",
  resave: false,
  saveUninitialized: false,
  cookie: {
    domain: "<domain_name>",
    maxAge: 7 * 24 * 60 * 60 * 1000,
    httpOnly: true,
    path: "/",
    sameSite: "none",
    secure: true
  }
}));

req.session

req.session는 응답 객체의 Session 객체에 접근하는 프로퍼티이다.

destroy(callback)

세션을 삭제하는 메서드로 완료되면 callback이 호출된다.

req.session.destroy(function(err){
  // cannot access session here
});

regenerate(callback)

세션을 재생성하는 메서드로 완료되면 req.session에서 세션 ID 및 세션 인스턴스가 초기화되고 callback이 호출된다.

req.session.regenerate(function(err){
  // will have a new session here
});

reload(callback)

세션 저장소에서 세션 데이터를 다시 가져오는 메서드로 req.session의 Session 객체를 다시 채운 후 callback이 호출된다.

req.session.reload(function(err){
  // session updated
});

save(callback)

세션을 세션 저장소에 다시 저장하는 메서드로 이 메소드는 세션 데이터가 변경된 HTTP 응답의 끝에서 자동으로 호출된다.
이 때문에 일반적으로 이 메서드를 호출할 필요가 없다.

req.session.save(function(err){
  // session saved
});
