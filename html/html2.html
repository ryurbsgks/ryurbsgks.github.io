자주 사용하는 HTML Tag(Head 영역)

1. <head>

<head> 태그는 기계가 식별할 수 있는 문서 정보(메타데이터)를 담는다.
정보로는 문서가 사용할 <title>, <script>, <style>, <link>, <meta>, <base> 태그 등이 있다.

<html>
  <head>
    <head 안에 들어가는 태그>
  </head>
</html>

2. <title>

<title> 태그는 브라우저의 제목 표시줄이나 페이지 탭에 보이는 문서 제목을 정의한다.

<head> 태그 안에는 하나의 <title> 태그만 존재해야 한다.

<head>
  <title>문서 제목</title>
</head>

3. <base>

<base> 태그는 문서 안의 모든 상대 URL이 사용할 기준 URL을 지정한다.

문서에는 하나의 <base> 태그만 존재할 수 있고 End tag가 존재하면 안된다.

<head>
  <base target="<target_value>" href="<URL>">
</head>

target

target 속성은 링크된 문서를 클릭했을 때 문서가 열릴 위치를 명시한다.

속성값으로는 _self, _blank, _parent, _top이 있으며 자주 쓰이는 속성값은 _blank이다.

_blank

링크된 문서를 새로운 브라우저에서 보여준다.

href

href 속성은 문서 내 상대 URL이 사용할 기준 URL을 속성값으로 갖는다.

4. <script>

<script> 태그는 데이터와 실행 가능한 코드를 문서에 포함할 때 사용하며 보통 JS코드와 함께 쓴다.

<head>
  <script src="<JS_file_path>"></script>
</head>

src

src 속성은 문서 내에 스크립트를 직접 삽입하는 것 대신 사용할 수 있다.

속성값으로는 외부 스크립트를 가리키는 URL로 절대와 상대 URL 모두 가능하다.

5. <style>

<style> 태그는 문서나 문서 일부에 대한 스타일 정보를 포함한다.

<head>
  <style>
    <style_css>
  </style>
</head>

<style> 태그는 문서의 <head> 태그안에 위치해야 하지만 일반적으로 스타일은 외부 스타일 시트에 작성하고 <link> 태그로 연결한다.

6. <link>

<link> 태그는 현재 문서와 외부 리소스의 관계를 명시하며 End tag가 존재하면 안된다.
주로 <link> 태그는 스타일 시트를 연결할 때 제일 많이 사용한다.

<head>
  <link rel="stylesheet" href="<CSS_file_path>">
</head>

rel

rel 속성은 연결할 리소스와 현재 문서의 관계를 나타낸다.

href

href 속성은 연결할 리소스의 URL로 절대와 상대 URL 모두 가능하다.

7. <meta>

<meta> 태그는 <base>, <link>, <script>, <style>, <title> 태그와 같은 다른 메타관련 태그로 나타낼 수 없는 메타데이터를 나타내며 End tag가 존재하면 안된다.

<head>
  <meta charset="utf-8">
  <!-- 3초 후 url로 리다이렉트 -->
  <meta http-equiv="refresh" content="3;url=<URL>">
</head>

charset

charset 속성은 해당 HTML 문서의 문자 인코딩 방식을 명시한다.
유니코드를 위한 문자셋인 UTF-8이 가장 많이 사용된다.

content

content 속성은 http-equiv 또는 name 속성의 값을 담는다.

name

naem 속성은 메타데이터를 위한 이름을 명시한다.

naem 속성이 명시되어 있다면 반드시 content 속성도 함께 명시되어야 한다.

http-equiv

http-equiv 속성은 content 속성에 명시된 값에 대한 HTTP 헤더를 제공한다.
http-equiv 속성이 명시되어 있다면 반드시 content 속성도 함께 명시되어야 한다.
