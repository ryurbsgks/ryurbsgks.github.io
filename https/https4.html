CSRF

1. CSRF

CSRF(Cross-Site Request Forgery)는 웹 애플리케이션의 취약점 중 하나로 사용자가 자신의 의지와 무관하게 공격자가 의도한 행동을 해서 특정 웹페이지를 보안에 취약하게 하거나 수정, 삭제 등의 작업을 하게 만드는 공격 방법이다.

공격자는 다른 사이트이기 때문에 응답에 해당하는 정보를 받을 수 없으므로 데이터에 직접 접근할 수 없다.
따라서 다른 사이트에서 사용자가 보내는 요청을 조작하여 공격자는 원하는 데이터를 탈취하거나 행동을 취할 수 있다.

2. CSRF 공격 조건

- 쿠키를 사용한 로그인을 할 때 쿠키로 해당 유저가 어떤 유저인지 알 수 있어야 한다.

- 공격자가 만든 피싱 사이트에 사용자가 접속을 해야한다.

- 해커가 요청에 모를 수 있는 정보가 담겨 있으면 공격할 수 없으므로 예측할 수 있는 요청과 파라미터를 가지고 있어야 한다.

3. CSRF 방어 방법

CSRF 방어 방법은 CSRF 공격 조건을 차단하면 된다.
방법에는 여러가지가 있지만 여기에서는 가장 간단한 아래 2가지 방법을 소개한다.

CSRF Token 토큰 사용

서버측에서 CSRF 공격에 보호하기 위한 토큰 문자열을 유저의 브라우저와 웹 애플리케이션에만 제공하여 사용자의 모든 요청에 대하여 서버에서 검증하는 방법이다.

Cookie의 Same-site 사용

Cookie의 Same-site를 사용하여 같은 도메인에서만 세션과 쿠키를 사용할 수 있게 한다.
