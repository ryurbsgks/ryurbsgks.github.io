Merge conflict & Upstream

1. Merge conflict

브랜치를 병합하거나 pull 과정 중 병합하는 과정에서 충돌이 일어나는 현상을 Merge conflict라고 한다.

충돌이 일어난 부분을 확인 후 수정을 하면 Merge conflict가 해결된다.

충돌이 발생한 파일을 에디터로 열어 보면 코드의 최상단에 수정 방법으로 Accept Current Change, Accept Incoming Change, Accept Both Changes가 제공된다.

이 3가지 수정 방법 외에 Compare Changes가 제공되는데 Compare Changes는 충돌한 파일을 각각 열어서 비교하는 옵션이다.

수정을 마치면 병합 커밋(merge commit)을 생성해 주기 위해서 파일을 Staging area로 추가해야 한다.

Accept Current Change

Accept Current Change는 내가 수정한 내용으로 파일에 반영하는 방법이다.

Accept Incoming Change

Accept Incoming Change는 Remote Repository의 내용으로 파일에 반영하는 방법이다.

Accept Both Changes

Accept Both Changes는 변경 사항 모두를 파일에 반영하는 방법이다.

2. Upstream과 Downstream

Upstream과 Downstream은 두개의 Repository간의 관계에 따라 정의되는 상대적인 개념이다.

예를들면 A, B Repository가 존재하고 A Repository로부터 B Repository로 pull from 해오고 B Repository에서 A Repository로 push to 한다면 A Repository는 Upstream이며 B Repository는 Downstream이 된다.

3. Upstream과 Origin

가장 근본이 되는 Repository를 Upstream이라고 부르고 내가 Fork해서 가져 온 Repository를 Origin이라고 부른다.
