자주 사용하는 HTML Tag(Body 영역)

1. <body>

<body> 태그는 HTML 문서의 내용을 나타내며 한 문서에 하나의 <body> 태그만 존재할 수 있다.

<html>
  <head>
  </head>
  <body>
    <body 안에 들어가는 태그>
  </body>
</html>

2. <div>

<div> 태그는 문서에서 특정 영역이나 구획을 정의할 때 사용하며 여러 요소들을 하나로 묶어 줄 수도 있다.

<div><div 영역에 들어갈 태그 or 내용></div>

3. <span>

<span> 태그는 그 자체만으로는 어떠한 의미도 가지지 않지만 스타일을 적용하거나 HTML 문서에서 인라인 요소들을 하나로 묶을 때 사용한다.

<span><span 영역에 들어갈 태그 or 내용></span>

<div>와 <span> 태그의 차이

<div>와 <span> 태그의 차이는 <div>는 블록 타입의 태그이고 <span>은 인라인 타입의 태그이다.

4. <h1 ~ h6>

<h1 ~ h6> 태그는 HTML 문서에서 제목을 정의할 때 사용하며 숫자가 작을 수록 텍스트의 크기가 더 크며 제목을 더 강조한다.

<h1>제목</h1>

5. <p>

<p> 태그는 하나의 문단을 나타낸다.

<p>문단</p>

6. <a>

<a> 태그는 href 속성을 통해 다른 페이지나 같은 페이지의 특정 위치로 이동할 수 있는 하이퍼링크를 만든다.

<a href="<URL>">하이퍼링크될 URL에 대한 설명</a>

href

href 속성은 하이퍼링크가 가리키는 URL을 속성값으로 갖는다.

7. <img>

<img> 태그는 문서에 이미지를 삽입할 때 사용한다.

<img src="<img_path>">

src

src 속성은 필수이며 포함하고자 하는 이미지의 경로를 지정한다.

alt

alt 속성은 필수는 아니지만 이미지를 표시할 수 없는 경우에 이 속성값을 대신 보여준다.

8. <form>

<form> 태그는 정보를 제출하기 위한 구획을 나타낸다.

<form>
  <form 안에 들어가는 태그>
</form>

9. <button>

<button> 태그는 클릭 가능한 버튼을 나타낸다.

<button type="<type_value>">button_name</button>

type

type 속성은 버튼의 행동 방식을 나타내는 속성으로 속성값으로는 submit, reset, button이 있다.

submit

버튼이 서버로 양식 데이터를 제출하며 default 값이다.

button

기본 행동이 없으며 클릭했을 때 아무것도 하지 않는다.
특정 행동을 하기 위해서는 클라이언트측 스크립트와 연결하여 사용한다.

reset

<button> 태그를 감싸고 있는 <form> 태그의 데이터를 초기화한다.

value

value 속성은 해당 버튼의 초기 값을 명시한다.

form

form 속성을 사용하면 버튼을 <form> 태그에 넣지 않고도 연결할 수 있으며 부모 중 form이 있더라더도 form을 재정의할 수 있다.

form 속성을 명시하지 않았으면 부모 중 <form> 태그가 존재하는 form과 연결된다.

disabled

disabled 속성을 명시하면 버튼을 누르거나 클릭하는 것을 막는다.

10. <input>

<input> 태그는 사용자로부터 입력을 받을 수 있는 입력 필드를 정의할 때 사용하며 End tag가 존재하면 안된다.
입력 필드는 type 속성의 값을 변경함으로써 여러 가지 입력 필드를 나타낼 수 있다.

<input type="<type_value>">

type의 속성값에는 수많은 값이 있지만 여기에서는 자주 사용하는 속성값만 다룬다.

button

기본 행동을 가지지 않으며 value 값을 레이블로 사용하는 버튼이다.

submit

양식을 전송하는 버튼이다.

text

default 값으로 한줄의 텍스트 필드를 제공한다.

number

숫자를 입력하기 위한 속성값이다.

password

값이 가려진 한줄 텍스트 필드를 제공한다.

email

이메일 주소를 입력할 수 있는 필드를 제공한다.

checkbox

단일 값을 선택하거나 선택 해제할 수 있는 체크박스이다.

radio

같은 name 값을 가진 여러개의 버튼 중에서 하나의 값을 선택하게 하는 버튼이다.

hidden

보이지 않지만 값은 서버로 전송할 수 있도록 하는 속성값이다.

date

날짜(년월일)를 지정할 수 있는 컨트롤이며 클릭시 날짜를 선택할 수 있는 달력을 열어준다.

file

클릭시 파일을 지정할 수 있는 컨트롤을 열어준다.

11. <textarea>

<textarea> 태그는 여러 줄의 텍스트를 입력할 수 있는 텍스트 입력 영역을 정의할 때 사용한다.

텍스트 입력 영역의 크기는 cols, rows 속성으로 지정할 수 있으며 CSS의 height, width 속성을 사용하면 더욱 손 쉽게 지정할 수 있다.
이때 cols, rows의 속성값은 양의 정수이다.

<textarea cols="<양의 정수>" rows="<양의 정수>"></textarea>

12. <li>

<li> 태그는 <ul> 또는 <ol> 태그 안에 들어가는 내용의 목록을 나타낼 때 사용한다.

<li> 태그의 부모 요소로는 반드시 <ul> 또는 <ol> 태그가 있어야 하며 <ul>, <ol> 태그는 중복해서 사용 가능하다.

<ol>
  <li>목록
    <ul>
      <li>목록</li>
    </ul>
  </li>
</ol>

13. <ul>

<ul> 태그는 순서가 없는 단순 목록을 표현할 때 사용한다.

<ul>
  <li>목록</li>
</ul>

14. <ol>

<ol> 태그는 순서가 있는 목록을 표현할 때 사용한다.

<ol>
  <li>목록</li>
</ol>

15. <dl>

<dl> 태그는 어떠한 단어 또는 정의에 대한 설명이 묶여있을 때 그 목록을 리스트 형식으로 나타낼 때 사용한다.

<dl>
  <dt>어떠한 단어 또는 정의</dt>
  <dd>해당 단어 또는 정의에 대한 설명</dd>
</dl>

<dl> 태그의 자식 태그로 <dt>, <dd> 태그가 존재한다.

<dt>

<dt> 태그는 어떠한 단어 또는 정의를 나타낸다.

<dd>

<dd> 태그는 해당 단어 또는 정의에 대한 설명을 나타낸다.

16. <table>

<table> 태그는 행과 열로 이루어진 표를 나타낸다.

<table>
  <tr>
    <th>열의 제목</th>
    <td>테이블 셀의 데이터</td>
  </tr>
</table>

<table> 태그의 자식 태그로 <tr>, <th>, <td> 태그가 존재한다.

<tr>

<tr> 태그는 테이블의 각 행을 정의한다.

<th>

<th> 태그는 테이블의 각 열의 제목을 정의한다.

<td>

<td> 태그는 하나의 테이블 셀을 정의한다.

17. <b>

<b> 태그는 텍스트를 시각적으로 강조하며 텍스트 자체를 강조하는 볼드체이다.

<b>강조할 텍스트</b>

18. <strong>

<strong> 태그는 텍스트를 시각적으로 강조하며 텍스트 자체를 강조하는 볼드체이다.

<strong>강조할 텍스트</strong>

<b>와 <strong> 태그의 차이

<b>와 <strong> 태그의 차이는 스크린 리더(컴퓨터의 화면 낭독 소프트웨어)에서의 차이가 크다.
<b> 태그는 강조하지 않지만 <strong> 태그는 강조해서 읽어준다.

19. <i>

<i> 태그는 텍스트를 이탤릭체로 표현한다.

<i>강조할 텍스트</i>

20. <em>

<em> 태그는 텍스트를 강조하는 이탤릭체를 표현한다.

<em>강조할 텍스트</em>

<i>와 <em> 태그의 차이

<i>와 <em> 태그의 차이는 스크린 리더(컴퓨터의 화면 낭독 소프트웨어)에서의 차이가 크다.
<i> 태그는 강조하지 않지만 <em> 태그는 강조해서 읽어준다.

21. <br>

<br> 태그는 텍스트 안에 줄바꿈을 할 때 사용하며 End tag가 존재하면 안된다.

<br>

22. <del>

<del> 태그는 문서에서 제거된 텍스트의 범위를 나타낸다.

<del>제거된 텍스트</del>

23. <code>

<code> 태그는 코드의 일부분을 나타낼 때 사용하며 CSS를 사용하여 더욱 다양한 모습으로 꾸밀 수 있다.

<code>작성할 code</code>

24. <header>

<header> 태그는 시멘틱 태그로 문서나 특정 섹션의 헤더를 정의할 때 사용한다.

<header>
  <header 안에 들어가는 태그>
</header>

25. <nav>

<nav> 태그는 시멘틱 태그로 다른 페이지 또는 현재 페이지의 다른 부분과 연결되는 네비게이션 링크들의 집합을 묶어두기 위한 요소이다.

<nav>
  <nav 안에 들어가는 태그>
</nav>

26. <section>

<section> 태그는 시멘틱 태그로 문서에 포함된 독립적인 섹션을 정의할 때 사용한다.

<section>
  <section 안에 들어가는 태그>
</section>

27. <main>

<main> 태그는 시멘틱 태그로 해당 문서의 주 컨텐츠를 정의할 때 사용한다.

<main>
  <main 안에 들어가는 태그>
</main>

28. <article>

<article> 태그는 시멘틱 태그로 문서, 페이지, 애플리케이션 또는 사이트 안에서 독립적으로 구분해 배포하거나 재사용할 수 있는 구획을 나타낸다.

<article>
  <article 안에 들어가는 태그>
</article>

29. <aside>

<aside> 태그는 시멘틱 태그로 페이지의 다른 컨텐츠들과 직접적인 관련이 없거나 약간의 연광성을 가지고 있지만 해당 컨테츠들로부터 분리시킬 수 있는 컨텐츠로 구성된 페이지 영역을 정의할 때 사용한다.

<aside>
  <aside 안에 들어가는 태그>
</aside>

30. <footer>

<footer> 태그는 시멘틱 태그로 문서나 특정 섹션의 하단부를 정의할 때 사용한다.

일반적으로 작성자, 저작권 정보, 관련 문서 등의 내용을 담는다.

<footer>
  <footer 안에 들어가는 태그>
</footer>
