Redux

1. Redux

Redux는 JS 앱을 위한 예측 가능한 상태 컨테이너 즉, JS 상태 관리 라이브러리이다.

Redux는 일관되게 작동하고 다양한 환경(클라이언트, 서버 등)에서 실행되며 테스트하기 쉬운 애플리케이션을 작성하는 데 도움이 된다.
또한 시간 여행 디버깅(time travel debugging)과 같은 훌륭한 개발자 경험을 제공한다.

Redux는 React 또는 다른 view 라이브러리와 함께 사용할 수 있다.

Redux의 장점

Redux는 상태를 예측 가능하게 만들어 주며 유지보수와 디버깅에 유리하다는 장점이 있다.

2. Redux 3가지 원칙

Redux의 기초를 이루는 3가지 원칙은 Single source of truth, State is read-only, Changes are made with pure functions 3가지 이다.

Single source of truth

애플리케이션의 전역 상태(global state)는 단일 저장소(single store) 내의 하나의 객체에 트리 구조로 저장된다.

State is read-only

상태를 변경하는 유일한 방법은 Action 객체를 전달하는 방법 뿐이다.

Changes are made with pure functions

Action에 의해 상태 트리가 어떻게 변화하는지를 지정하기 위해서는 순수 Reducer를 작성해야한다.

Reducer는 이전 상태와 Action을 받아 다음 상태를 반환하는 순수 함수이다.

3. Redux 설치

Node 애플리케이션에서 Redux를 설치하기 위해 npm install을 이용한다.

npm install redux

Redux toolkit 설치

Redux toolkit 설치 방법은 아래와 같다.

npm install @reduxjs/toolkit

Redux toolkit은 Redux를 사용하기 위한 공식 권장 방식이다.
Redux toolkit은 Redux 앱을 빌드하는데 필수적이라고 생각하는 패키지와 기능이 포함되어 있다.

Redux toolkit에 대한 내용은 따로 다루지 않고 Redux toolkit 공식문서에 대한 링크만 남긴다.

4. Redux 용어

Redux의 대표적인 Store, Action, Dispatch, Reducer 용어에 대해서 설명한다.

Store

Store는 응용 프로그램의 상태 트리를 보유하는 객체로 Redux 앱에는 단일 Store만 있어야 한다.

Action

Action은 상태를 변경 하려는 의도를 나타내는 일반 객체로 데이터를 Store로 가져오는 유일한 방법이다.

Action에는 수행 중인 작업의 유형을 나타내는 type 필드가 있어야 한다.

Dispatch

Dispatch 함수는 Action이나 비동기 Action을 받는 함수로 받은 다음 하나나 여러개의 Action을 Store에 보낼수도 보내지 않을 수도 있다.

기본 Dispatch 함수는 반드시 동기적으로 Store의 Reducer에 Action을 보내야 한다.
그러면 Reducer는 Store가 반환한 이전 상태와 함께 새 상태를 계산한다.

Reducer

Reducer는 누적값과 값을 받아서 새로운 누적값을 반환하는 함수이다.

Redux에서 누적값은 상태 객체이고 누적될 값은 Action이다.
Reducer는 주어진 이전 상태와 Action에서 새로운 상태를 계산하는 순수 함수이다.

Reducer의 Immutability

Reducer의 Immutability(불변성)는 Redux의 상태 업데이트는 immutable한 방식으로 변경해야 한다는 것이다.
즉, Reducer는 Side Effect를 가져서는 안된다.
이를 통해 Redux의 장점 중 하나인 핫 리로딩(hot reloading)과 시간 여행 디버깅(time travel debugging)과 같은 기능들이 가능해진다.
