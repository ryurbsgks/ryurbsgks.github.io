ELB 생성 & ACM을 통한 인증서 발급

1. ELB 생성 & ACM을 통한 인증서 발급

AWS 메인 콘솔 창에서 EC2를 검색하여 EC2 메인 콘솔로 이동한다.

로드밸런서를 클릭한다.

Load Balancer 생성 버튼을 클릭한다.

Load Balancer 유형 선택에서 Application Load Balancer의 생성 버튼을 클릭한다.

Load Balancer 구성에서 기본 구성 옵션에 ELB 이름을 설정하고 리스너 옵션에 프로토콜을 추가한다.

Load Balancer 구성에서 가용 영역 옵션에서 아래 사진과 같이 선택 후 다음: 보안 설정 구성 버튼을 클릭한다.

ACM으로부터 새 인증서 요청을 클릭한다.

도메인 이름 입력 후 다음 버튼을 클릭한다.

아래 사진과 같이 다음 버튼을 클릭한다.

아래 사진과 같이 검토 버튼을 클릭한다.

아래 사진과 같이 확인 및 요청 버튼을 클릭한다.

Route 53에서 레코드 생성 버튼을 클릭한다.

아래 사진과 같이 화면이 뜨면 생성 버튼을 클릭한다.

레코드가 생성 되어서 성공 문구가 뜨면 계속 버튼을 클릭한다.

인증서 발급까지 시간이 걸릴 수 있어 그 전까지는 아래의 사진과 같이 인증서 요청의 상태가 검증 보류 상태이다.

인증서 발급이 완료되면 아래 사진과 같은 화면이 보인다.

Load Balancer 생성 창으로 돌아가 아래 사진에 표시된 recurring 아이콘을 클릭한다.

인증서 선택 후 다음: 보안 그룹 구성 버튼을 클릭한다.

보안 그룹 선택 후 다음: 라우팅 구성 버튼을 클릭한다.

대상 그룹 이름을 설정한다.

고급 상태 검사 설정 토글을 클릭한다.

성공 코드를 201번으로 수정 후 다음: 대상 등록 버튼을 클릭한다.

EC2 인스턴스 체크 표시 후 등록된 항목에 추가 버튼을 클릭한다.

선택한 EC2 인스턴스의 등록 여부 확인 후 다음: 검토 버튼을 클릭한다.

요약된 내용을 검토 후 생성 버튼을 클릭한다.

생성 완료 화면이 정상적으로 나오면 닫기 버튼을 클릭한다.

로드밸런서 메뉴로 돌아가 DNS 이름 주소를 이용하여 접속 테스트를 진행한다.
