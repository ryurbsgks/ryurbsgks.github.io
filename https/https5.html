Cookie

1. Cookie

Cookie(쿠키)는 서버가 웹 브라우저에 정보를 저장하고 불러올 수 있는 수단이며 해당 도메인에 대한 Cookie가 존재하면 웹 브라우저는 도메인에게 http 요청 시 Cookie를 함께 전달한다.

Cookie는 응답 헤더에 심어서 전송되며 이때 전달되는 Cookie는 key-value 형식의 데이터이다.
value는 문자열 또는 JSON으로 변환된 객체일 수 있다.

2. Cookie Options

서버는 쿠키를 이용하여 데이터를 저장하고 원할 때 이 데이터를 다시 불러와 사용할 수 있지만 데이터를 저장한 이후 아무 때나 데이터를 가져올 수 없다.
데이터를 다시 가져오기 위해서는 특정 조건들이 만족해야 하는데 이런 조건들은 Cookie Options(쿠키 옵션)으로 표현할 수 있다.

Domain

쿠키 옵션에서 도메인 정보가 존재한다면 클라이언트에서는 쿠키의 도메인 옵션과 서버의 도메인이 일치해야만 쿠키를 전송할 수 있다.

Path

Path 옵션의 특징은 설정된 path를 전부 만족하는 경우 요청하는 Path가 추가로 더 존재하더라도 쿠키를 서버에 전송할 수 있다.

MaxAge & Expires

쿠키가 유효한 기간을 정하는 옵션이다.

MaxAge는 앞으로 몇 초 동안 쿠키가 유효한지 설정하는 옵션이다.

Expires는 언제까지 쿠키가 유효한지 Date를 지정하는 옵션이다.
이때 클라이언트의 시간을 기준으로 한다.

이후 지정된 시간, 날짜를 초과하게 되면 쿠키는 자동으로 파괴된다.

만약 두 옵션이 모두 지정되지 않는 경우에는 브라우저의 탭을 닫아야만 쿠키가 제거된다.

HttpOnly

자바스크립트에서 브라우저의 쿠키에 접근 여부를 결정한다.

default 값은 false이며 해당 옵션이 true면 자바스크립트에서는 쿠키에 접근이 불가하다.

Secure

쿠키를 전송해야 할 때 사용하는 프로토콜에 따른 쿠키 전송 여부를 결정한다.

해당 옵션이 true면 HTTPS 프로토콜을 이용하여 통신하는 경우에만 쿠키를 전송할 수 있다.

SameSite

CORS요청을 받은 경우 요청에서 사용한 메소드와 해당 옵션의 조합으로 서버의 쿠키 전송 여부를 결정하게 된다.

SameSite의 사용 가능한 옵션은 Lax, Strict, None이 있다.

Lax

CORS요청이면 GET 메소드에 대해서만 쿠키를 전송할 수 있다.

Strict

CORS요청이 아닌 요청을 보낸 Origin과 서버의 도메인이 같은 경우에만 쿠키를 전송 할 수 있다.

None

항상 쿠키를 보내줄 수 있다.(단, 쿠키 옵션 중 Secure 옵션이 필요하다)

3. Cookie 사용방법

서버에서 클라이언트로 쿠키를 처음 전송하게 된다면 헤더에 Set-Cookie라는 프로퍼티에 쿠키를 담아 쿠키를 전송한다.
이후 클라이언트 혹은 서버에서 쿠키를 전송해야 한다면 클라이언트는 헤더에 cookie라는 프로퍼티에 쿠키를 담아 서버에 쿠키를 전송한다.

쿠키를 사용하는 방법은 여러가지가 있지만 여기에서는 Express를 이용한 쿠키 사용방법만 다룬다.

res.cookie()

res.cookie() 메서드는 제공된 옵션으로 HTTP Set-Cookie 헤더를 설정하는 함수이다.

res.cookie(<cookie_key>, <cookie_value>)

Cookie에 Option이 있다면 객체의 형태로 추가해준다.

res.cookie(<cookie_key>, <cookie_value>, {path: "/", httpOnly: true})

res.clearCookie()

res.clearCookie() 메서드는 지정된 쿠키를 지우는 함수이다.
주의할 점은 Expires와 MaxAge를 제외하고 주어진 옵션이 res.cookie()에 제공된 옵션과 동일한 경우에만 쿠키를 지운다.

Optiont 사용방법은 res.cookie()와 같다.

res.clearCookie(<cookie_key>, {<Options>})

req.cookies

req.cookies는 요청에서 보낸 쿠키가 포함된 객체이며 요청에 쿠키가 포함되어 있지 않으면 default는 "{}"이다.
