Express Router

1. 라우팅

클라이언트는 특정한 HTTP 요청 메서드(GET, POST 등)로 서버의 특정 URI(또는 경로)로 HTTP 요청을 보낸다.
라우팅은 클라이언트의 요청에 해당하는 메서드와 Endpoint에 따라 서버가 응답하는 방법을 결정하는 것을 말한다.

2. Express Router

Express에서는 라우팅 기능을 Router Middleware로 제공하고 있다.

express.Router 클래스를 사용하여 router 인스턴스를 생성할 수 있다.
생성된 router 인스턴스는 라우팅, Middleware 기능을 사용할 수 있으며 모듈화 할 수 있다.

const express = require('express')
const router = express.Router()

Example

공식문서에 나와있는 예시로 brids.js라는 router 기능의 파일을 만들고 모듈화하여 앱에서 router 모듈을 로드하는 예시이다.

birds.js

const express = require('express')
const router = express.Router()

router.use((req, res, next) => {
  console.log('Time: ', Date.now())
  next()
})

router.get('/', (req, res) => {
  res.send('Birds home page')
})

router.get('/about', (req, res) => {
  res.send('About birds')
})

module.exports = router

app.js

const birds = require('./birds')

<app.js에서 동작할 코드>

app.use('/birds', birds)
