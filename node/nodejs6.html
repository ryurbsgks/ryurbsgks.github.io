http 모듈

1. http 모듈

http 모듈은 Node.js의 가장 기본적인 웹 모듈로 HTTP 웹 서버를 생성하는 것과 관련된 모든 기능을 담당한다.

http 모듈 불러오기

아래와 같이 require() 함수를 사용하여 http 모듈을 불러올 수 있다.

const http = require('http')

2. http 모듈 메서드

http 모듈에는 많은 메서드가 내장되어 있지만 여기에서는 Server 객체를 만드는 createServer(), Server 객체, ServerResponse 객체의 메서드들만 다룬다.
더 많은 http 모듈 메서드에 대해서 알고 싶다면 http 모듈 공식 API 문서를 참고하자

http.createServer()

http.createServer() 메서드는 서버 객체를 생성하는 메서드로 생성된 서버 객체는 EventEmitter이다.

http.createServer() 메서드는 requestListener를 매개변수로 받을 수 있다.

const server = http.createServer(<requestListener>)

or

const server = http.createServer( (request, response) => {
  <동작할 코드>
})

requestListener

requestListener는 서버가 요청을 받을 때마다 호출되는 함수로 사용자의 요청과 응답을 처리한다.

function (request, response) {
  <동작할 코드>
}

Server 객체

http.createServer() 메서드로 생성된 Server 객체의 메서드로 listen(), close() 메서드에 대해서 다룬다.

server.listen()

server.listen() 메서드는 지정된 port에 연결을 수신하는 서버를 시작한다.

server.listen(<port>, <callback>)

<port>는 시작하는 서버의 port 번호로 생략되거나 0이면 사용하지 않는 임의의 port를 할당한다.

<callback>은 listen() 메서드에서 사용할 callback 함수로 생략 가능하다.

server.close()

server.close() 메서드는 서버가 새 연결을 수락하지 못하도록 중지한다.

server.close(<callback>)

<callback>은 서버가 닫힐 때 호출되는 callback 함수로 생략 가능하다.

ServerResponse 객체

생성된 Server 객체의 response 객체의 메서드로 writeHead(), write(), end() 메서드에 대해서 다룬다.

response.writeHead()

response.writeHead() 메서드는 요청에 대한 응답 헤더를 보낸다.

response.writeHead() 메서드는 한 번만 호출되어야 하며 response.end()가 호출되기 전에 호출되어야 한다.

response.writeHead(<statusCode>, <statusMessage>, <headers>)

<statusCode>는 HTTP 상태 코드를 입력한다.

<statusMessage>는 문자열로 상태 메세지를 입력하며 생략 가능하다.

<headers>는 응답 헤더로 객체나 배열로 입력하며 생략 가능하다.

response.write()

response.write() 메서드는 요청에 대한 응답 본문을 보낸다.

response.write() 메서드는 response.writeHead() 메서드와 달리 여러번 호출될 수 있다.

response.write(<chunk>, <encoding>, <callback>)

<chunk>는 응답 본문에 보낼 문자열 또는 버퍼이다.

<encoding>은 생략 가능하며 생략 시 default 값은 utf8 이며 <chunk>가 문자열인 경우 <encoding>은 byte stream으로 인코딩하는 방법을 지정한다.

<callback>은 <chunk>가 보내질 때 호출되며 생략 가능하다.

response.end()

response.end() 메서드는 모든 응답 헤더와 본문이 전송되었음을 서버에 알린다.

response.end(<data>, <callback>)

<data>는 생략 가능하며 <data> 값이 존재하면 response.write(<data>) 다음에 response.end()를 호출하는 것과 사실상 유사하다.

<callback>은 응답이 완료될 때 호출되며 생략 가능하다.
