MongoDB

1. MongoDB

MongoDB는 대표적인 NoSQL 도큐먼트 데이터베이스이다.

도큐먼트 데이터베이스는 데이터를 테이블이 아닌 문서처럼 저장하는 데이터베이스를 의미한다.
일반적으로 도큐먼트 데이터베이스에서는 JSON 유사 형식으로 데이터를 문서화한다.

도큐먼트는 데이터를 필드-값의 형태로 가지고 있고 컬렉션이라고 하는 그룹으로 묶어서 관리한다.
데이터베이스는 여러 개의 컬렉션으로 구성된다.

컬렉션(Collection)

MongoDB의 도큐먼트로 구성된 저장소이다.

데이터베이스 당 많은 컬렉션이 있고 컬렉션 당 많은 도큐먼트가 있을 수 있다.

2. Atlas Cloud

MongoDB에서는 아틀라스(Atlas)로 클라우드에 데이터베이스를 설정한다.

아틀라스는 GUI와 CLI로 데이터를 시각화, 분석, 내보내기, 그리고 빌드하는 데에 사용할 수 있다.

아틀라스 사용자는 클러스터를 배포할 수 있으며 클러스터는 그룹화된 서버에 데이터를 저장한다.
이 서버는 레플리카 세트로 구성되어 있으며 레플리카 세트는 동일한 데이터를 저장하는 몇 개의 연결된 MongoDB 인스턴스의 모음이다.

도큐먼트나 컬렉션을 변경할 경우 변경된 데이터의 중복 사본이 레플리카 세트에 저장된다.

레플리카 세트

동일한 데이터를 저장하는 소수의 연결된 머신을 뜻한다.

레플리카 세트 중 하나에 문제가 발생하더라도 데이터를 그대로 유지할 수 있다.

인스턴스

로컬 또는 클라우드에서 특정 소프트웨어를 실행하는 단일 머신을 말한다.
MongoDB에서는 데이터베이스를 말한다.

클러스터

데이터를 저장하는 서버 그룹으로 여러 대의 컴퓨터를 네트워크를 통해 연결하여 하나의 단일 컴퓨터처럼 동작하도록 제작한 컴퓨터를 말한다.

3. MongoDB 데이터 포맷 형식(JSON, BSON)

shell을 이용하여 도큐먼트를 조회하거나 업데이트할 때 도큐먼트는 JSON형식으로 출력된다.

MongoDB는 JSON 형식으로 작성된 것은 무엇이든 데이터베이스에 추가할 수 있고 쉽게 조회할 수 있다.
그러나 그 내부에서는 속도, 효율성, 유연성의 장점이 있는 BSON으로 데이터를 저장하여 사용한다.

도큐먼트는 모두 중괄호({})로 시작하고 끝나며 각 필드와 값의 쌍은 쉼표(,)로 구분된다.
필드와 값의 사이에는 콜론(:)이 작성되어 있고 각 필드는 쌍따옴표("")를 사용하여 문자열로 표현한다.
이 조건 중 어느 하나라도 충족되지 않는다면 이는 유효한 도큐먼트가 아니다.

JSON

JSON은 텍스트 형식이기 때문에 읽기 쉽고 많은 개발자들이 사용하기 편리한 형태를 가지고 있어 데이터를 저장하는 좋은 방법 중 하나이다.
하지만 JSON은 파싱이 느리고 메모리 사용이 비효율적이라는 단점이 있으며 기본 데이터 타입만을 지원하기 때문에 사용할 수 있는 데이터 타입에 제약이 있다.

BSON

BSON은 컴퓨터의 언어에 가까운 이진법에 기반을 둔 표현법으로 JSON 보다 메모리 사용이 효율적이며 빠르고, 가볍고, 유연하며 더 많은 데이터 타입을 사용할 수 있다.

4. Importing & Exporting

MongoDB의 데이터는 단순히 백업 저장을 하기 위해서라면 가볍고 빠른 BSON의 형태를 사용하며 데이터를 내보낸 후 조회를 하거나 출력을 해야 한다면 사람이 읽기 쉬운 JSON의 형식을 사용한다.
즉, MongoDB의 데이터는 BSON의 형태로 저장이 되고 보통 읽기 쉬운 JSON의 형태로 출력된다.
그래서 JSON과 BSON에 따라 가져오거나 내보낼 때 사용 가능한 명령어가 각각 존재한다.

Importing

데이터를 가져오기 위해서는 Atlas Cluster URI가 필요하다.
해당 URI는 일반 웹의 URI와 형식이 같고 username, password, cluster 주소로 이루어져 있다.

기존에 있는 데이터를 삭제하기 위한 옵션인 drop 쿼리문은 선택적으로 사용할 수 있다.

mongoimport

JSON 형식으로 데이터를 가져오기 위한 명령어

mongoimport --uri "<AtlasClusterURI>" --drop=<file_name>.json

mongorestore

BSON 형식으로 데이터를 가져오기 위한 명령어

mongorestore --uri "<AtlasClusterURI>" --drop dump

Exporting

Importing과 마찬가지로 Atlas Cluster URI를 사용해서 작성한다.

mongoexport

JSON 형식으로 데이터를 내보내기 위한 명령어

mongoexport --uri "<AtlasClusterURI>" --collection=<collection_name> --out=<file_name>.json

mongodump

BSON 형식으로 데이터를 내보내기 위한 명령어

mongodump --uri "<AtlasClusterURI>"
