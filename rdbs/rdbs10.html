MongoDB Methods

1. _id

모든 MongoDB 도큐먼트는 _id 필드를 기본값으로 반드시 가지고 있어야 한다는 공통점이 있다.
이 _id 필드의 값은 각 도큐먼트를 구별하는 역할을 한다.

도큐먼트 내부의 필드와 값이 똑같다 할지라도 _id 값이 다르면 서로 다른 도큐먼트로 간주한다.
반대로 도큐먼트 내 필드와 값이 다르다고 하더라도 _id 값이 같다고 하면 서로 같은 도큐먼트로 여겨 에러를 발생한다.

도큐먼트를 추가할 때 _id 필드와 값을 특정하지 않았다면 자동적으로 _id 필드가 생성되고 값에 ObjectId 타입이 할당된다.

2. show

show 명령어는 존재하는 데이터베이스나 컬렉션을 확인하는 명령어이다.

show dbs show collections

3. use

use 명령어는 해당 데이터베이스를 사용하는 명령어이다.

use <database_name>

4. insert

새로운 도큐먼트를 추가하는 방법으로 도큐먼트를 해당 컬렉션에 삽입하는 명령어이다.
insert()의 괄호 안에 삽입하고자 하는 도큐먼트를 작성한다.

db.<collection>.insert(<document>, <ordered>)

한 번에 다수의 도큐먼트를 삽입하기 위해서는 배열 안에 해당하는 도큐먼트를 담아서 insert()의 괄호 안에 넣는다.
삽입 순서는 배열 안의 리스트 된 순서대로 진행된다.
만약 duplicate key 에러가 나오면 작업은 중단된다.

순서에 상관없이 도큐먼트를 추가하고 싶을 때는 ordered 매개변수를 false로 추가하면 순서에 상관 없이 고유한 _id를 가진 도큐먼트를 컬렉션에 삽입한다.
orderd 매개변수는 boolean타입으로 defaults값은 true로 생략가능하다.

존재하지 않는 컬렉션에 도큐먼트를 삽입하면 컬렉션이 만들어지는 동시에 도큐먼트가 해당 컬렉션에 삽입된다.

5. find

데이터를 조회하는 명령어로 find()의 괄호 안에 조회하고자 하는 조건을 쿼리문으로 작성한다.

db.<collection>.find(<query>)

해당 결과물은 JSON 형식으로 화면에 출력되며 랜덤하게 선택된 20개의 결과물만 출력된다.
결과물이 20개가 초과되어 다음 20개의 도큐먼트를 조회하기 위해서는 it 명령어를 사용해야 한다.

만약 해당 데이터베이스의 모든 데이터를 조회하고 싶다면 find 명령어를 쿼리문 없이 사용하면 된다.

6. findOne

조건을 만족하는 첫번째 데이터를 조회하는 명령어이다.

db.<collection>.findOne(<query>, <projection>)

projection 매개변수로 필드를 지정하면 해당 필드를 반환하며 _id 필드는 제외하지 않으면 항상 포함되서 반환한다.
만약 조건에 만족하는 첫번째 데이터의 모든 필드를 반환하고 싶으면 projection 매개변수를 생략하면 된다.

7. count

데이터의 수를 조회하기 위한 명령어이다.

db.<collection>.count(<query>)

find 명령어와 같이 사용한다면 아래와 같다.

db.<collection>.find(<query>).count()

8. update

mongo shell에서 도큐먼트를 업데이트하기 위한 방법에는 updateMany, updateOne 2가지가 있다.

updateMany

쿼리문과 일치하는 모든 도큐먼트를 업데이트하는 명령어

db.<collection>.updateMany(<query>, <update>)

update 매개변수에는 적용할 수정 사항을 입력한다.
이때 MQL(MongoDB Query Language) 업데이트 연산자를 이용하여 수정 사항을 작성한다.

updateOne

주어진 기준에 맞는 다수의 도큐먼트 중 첫번째 도큐먼트 하나만 업데이트하는 명령어이다.
사용방법은 updateMany와 같지만 차이점은 업데이트하는 데이터의 수 뿐이다.

db.<collection>.updateOne(<query>, <update>)

9. delete

mongo shell을 사용하여 도큐먼트 및 컬렉션을 삭제할 때 deleteOne과 deleteMany를 사용할 수 있다.

deleteOne

주어진 기준에 맞는 다수의 도큐먼트 중 첫번째 도큐먼트 하나를 삭제하는 명령어

db.<collection>.deleteOne(<query>)

deleteMany

쿼리문과 일치하는 모든 도큐먼트를 삭제하는 명령어

db.<collection>.deleteMany(<query>)

10. drop

데이터베이스에서 컬렉션을 삭제하기 위한 명령어

db.<collection>.drop()
