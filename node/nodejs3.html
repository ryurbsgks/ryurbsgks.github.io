Node.js 모듈

1. 모듈(module)

모듈이란 여러 기능들에 관한 코드가 모여있는 하나의 파일을 의미한다.

2. 3rd-party 모듈

3rd-party(서드 파티) 모듈은 해당 프로그래밍 언어에서 공식적으로 제공하는 빌트인 모듈(built-in module)이 아닌 모든 외부 모듈을 일컫는다.

3rd-party 모듈을 다운로드하기 위해서는 npm install 후 require 구문을 이용하여 해당 모듈을 불러온다.

npm install <module> const <moduleName> = require('<module>')

3. fs 모듈

fs(File System) 모듈은 파일을 읽거나 저장하는 등의 일을 할 수 있게 도와주는 모듈이다.

fs 모듈 불러오기

Node.js에서는 JS 코드 가장 상단에 require 구문을 이용하여 다른 파일을 불러올 수 있다.
fs 모듈 또한 아래와 같이 require 구문을 사용하여 불러올 수 있다.

const fs = require('fs')

fs 모듈 메서드

fs 모듈에는 많은 메서드가 내장되어 있지만 여기에서는 readFile(), writeFile() 메서드만 다룬다.
더 많은 모듈에 대해서 알고 싶다면 fs 모듈 공식 API 문서를 참고하자

fs.readFile()

readFile() 메서드는 파일의 전체 내용을 비동기적으로 읽는 메서드이다.

fs.readFile("<path>", <options>, <callback>)

<path>에는 읽으려는 파일의 경로와 파일명을 입력한다.

<options>는 객체 형태 또는 문자열이 올 수 있으며 주로 "utf8"을 사용한다.
<options>는 생략할 수 있으며 생략시 기본값은 utf8 이다.

<callback>은 파일을 읽고 난 후에 비동기적으로 실행되는 콜백 함수로 err, data 두 개의 매개변수가 존재한다.
에러가 발생하지 않으면 err는 null이 되며 data에 문자열이나 Buffer라는 객체가 전달된다.

fs.writeFile()

writeFile() 메서드는 주어진 데이터를 주어진 파일명으로 파일을 비동기적으로 만들어주는 메서드이다.
만약 해당 파일이 존재하면 파일 내용을 수정한다.

fs.writeFile("<file>", <data>, <options>, <callback>)

<path>에는 만드려는 파일의 경로와 파일명, 확장자명을 입력한다.

<data>는 파일을 만들때 입력할 데이터로 문자열 또는 Buffer가 올 수 있다.

<options>는 객체 형태 또는 문자열이 올 수 있으며 주로 "utf8"을 사용한다.
<options>는 생략할 수 있으며 생략시 기본값은 utf8 이다.

<callback>은 err 매개변수를 가지며 에러가 발생하였을 때 처리할 콜백 함수이다.
